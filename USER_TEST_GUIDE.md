# 🧪 Vibe Coder ユーザーテストガイド

## 🎯 テスト目的

Vibe Coder のユーザビリティと機能性を評価し、改善点を特定するためのユーザーテストです。

## 📋 事前準備

### 必要な環境
- **Chrome ブラウザ** (推奨)
- **スマートフォンまたはタブレット** (モバイル体験確認用)
- **マイク機能** (音声コマンドテスト用)

### 開発者による準備
```bash
# 1. ローカルテスト環境起動
npm run test:local

# 2. すべてのサービスが起動するのを待つ
# - シグナリングサーバー: localhost:3001
# - ホストサーバー: localhost:8080
# - PWA: vibe-coder.space または localhost:3000
```

## 🚀 テスト手順

### ステップ 1: 接続テスト

1. **PWA にアクセス**
   - Chrome で https://vibe-coder.space を開く
   - またはローカル環境の場合: http://localhost:3000

2. **Server ID を入力**
   - 開発者から提供された Server ID を入力
   - 「接続」ボタンをクリック

3. **接続確認**
   - 「接続済み」表示を確認
   - 接続にかかった時間を意識

### ステップ 2: 音声コマンドテスト

1. **マイクボタンをタップ**
   - 🎤 アイコンをタップ
   - マイク許可が求められたら「許可」

2. **音声コマンドを試す**
   ```
   「認証機能を追加して」
   「バグを修正して」
   「新しいファイルを作成して」
   「このコードをレビューして」
   ```

3. **音声認識の確認**
   - 音声が正しく認識されるか
   - Claude Code が適切に実行されるか

### ステップ 3: クイックコマンドテスト

1. **アイコンボタンを試す**
   - 🔐 Login: 認証機能追加
   - 🐛 Bug Fix: バグ修正
   - 🧪 Test: テスト実行
   - 🚀 Deploy: デプロイ準備

2. **ワンタップ実行を確認**
   - タップ後の反応速度
   - 実行結果の表示

### ステップ 4: モバイル体験テスト

1. **スマートフォンでアクセス**
   - https://vibe-coder.space をスマホで開く
   - 同じ Server ID で接続

2. **モバイル操作を確認**
   - タッチ操作の反応性
   - 画面サイズでの見やすさ
   - 音声入力の使いやすさ

### ステップ 5: プレイリスト機能テスト

1. **設定メニューを開く**
   - ⚙️ Settings アイコンをタップ

2. **プレイリスト発見**
   - 「Discover Playlists」をタップ
   - 利用可能なプレイリストを確認

3. **プレイリストインポート**
   - 気になるプレイリストをインポート
   - 新しいコマンドが追加されることを確認

## 📝 評価ポイント

### 接続体験
- [ ] 接続手順は分かりやすいか？
- [ ] 接続時間は許容範囲か？
- [ ] エラーなく接続できたか？

### 音声コマンド
- [ ] 音声認識の精度は十分か？
- [ ] 自然言語での指示は直感的か？
- [ ] マイクボタンは分かりやすいか？

### クイックコマンド
- [ ] アイコンの意味は理解しやすいか？
- [ ] ワンタップ操作は便利か？
- [ ] 反応速度は満足できるか？

### モバイル体験
- [ ] スマホでの操作は快適か？
- [ ] 画面レイアウトは適切か？
- [ ] タッチ操作に問題はないか？

### 全体的な体験
- [ ] 使い方は直感的か？
- [ ] パフォーマンスは満足できるか？
- [ ] 日常的に使いたいと思うか？


## 🔍 トラブルシューティング

### 接続できない場合
- Server ID が正しく入力されているか確認
- ネットワーク接続を確認
- ブラウザをリフレッシュして再試行

### 音声認識がうまくいかない場合
- マイクの許可設定を確認
- 静かな環境で再試行
- はっきりと話す

### モバイルで表示が崩れる場合
- ブラウザのズーム設定を確認
- 画面を縦向きで使用
- 他のタブを閉じる

## 📊 フィードバック収集

テスト完了後、以下のコマンドでフィードバックを収集：

```bash
npm run feedback
```

### 収集する情報
- **定量評価**: 各機能の満足度 (1-5 スケール)
- **定性評価**: 改善提案、使いにくかった点
- **利用意向**: 継続利用の意向、推薦度
- **デバイス情報**: 使用したデバイス、ブラウザ
